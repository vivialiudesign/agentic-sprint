# Writing Guide

Content principles applied by the Content Agent at every phase of
the sprint. These aren't sprint-specific — they're the baseline for any user-facing
copy an agent writes: wireframe labels, screen copy, HMW questions, principle
statements, and every Phase 5 output.

---

## Core rules

**Write for the reader, not for the room.** Copy inside a sprint should read the same
as copy that will ship — no internal jargon, no "TBD," no meta-commentary the end
user would never see.

**Real copy only — never placeholder text.** "Lorem ipsum," "[headline here]," and
generic filler are forbidden anywhere a human or end user will look, including
wireframes. If the exact final copy isn't decided yet, write a specific, plausible
draft — not a placeholder.

**Specific beats vague.** Replace "improve the experience" with what actually
improves and how. Replace "engaging" with what the reader will do differently. A
principle, HMW, or screen label that could apply to any product isn't sharp enough yet.

**One idea per sentence.** If a sentence needs "and" to hold two claims, it's two
sentences.

**Say the number, not the adjective.** "Fast" is not a metric. "Under 200ms" is.

**Match tone to the confirmed design principles.** If a design principle says "calm
not cluttered," the copy should be calm — short lines, no exclamation points, no
urgency it hasn't earned.

**Cut hedging.** "It seems like this might potentially help" → "This helps." If
genuine uncertainty exists, say so plainly with a confidence rating (see Source
Tracing in SKILL.md) — don't bury it in hedge words.

---

## Applying this per phase

| Phase | What to tighten |
|---|---|
| Define | Problem statement and principle wording — no vague adjectives, one trade-off per principle |
| Discover | Insight and HMW phrasing — specific claims, sourced, no hedging |
| Diverge | Wireframe copy — every idea's screen must show real, considered copy |
| Converge | The user story and every key screen's copy — this is what ships if the direction is approved |
| Build | Full copy pass on all five outputs before they reach the human |

---

## Quick self-check before shipping any copy

1. Would I be embarrassed if this shipped verbatim? If yes, it's not done.
2. Could I delete a word without losing meaning? If yes, delete it.
3. Does a stranger understand this with zero added context?
4. Does the tone match the principles we committed to?
